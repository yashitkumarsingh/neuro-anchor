📖 Neuro-Anchor Usage Guide

        
      


      

        

          
What is Neuro-Anchor?

          

            Neuro-Anchor is a cognitive accessibility layer designed to support neurodivergent software engineers (ADHD, Autism, Dyslexia, and others) by reducing cognitive load, buffering distractions, deconstructing vague specs, and helping context recovery.
          

        


        

          
Key Features

          
    
            
  • Cognitive Profiles: Modify the IDE environment sensory factors. Standard Mode, Focus Mode (hides sidebars/minimaps), and Low Sensory Mode (mutes squiggles/animations).
    • 
            
  • Interruption Shield: Toggles DND, buffers notifications, starts a Pomodoro timer, and stores interruptions in a list to look at later.
    • 
            
  • Ticket compiler: Decodes complex, vague descriptions into bite-sized step-by-step tasks so you don't get paralyzed figuring out how to start.
    • 
            
  • Context checkpoint: Queries Git and recent terminal commands to formulate a "What was I doing?" recovery summary to help restart when context-switched.
    • 
            
  • PR Feedback Translator: Translates passive-aggressive or vague code reviews into clear, actionable steps and provides replies.
    • 
          

        


        

          
How to run the CLI Options

          
If you prefer using the terminal command line directly, the following commands are available:

          
# Reconstruct current state summary
neuro-anchor status

# Compile ticket with specific cognitive granularity
neuro-anchor compile "Ticket title/body" --mode 5m

# Unpack confusing pull request remarks
neuro-anchor translate "PR feedback comment"

# Turn interruption shield on or off
neuro-anchor shield on
neuro-anchor shield off

        


        

          
Local Execution Whitelist

          

            For safety, only specific commands are whitelisted for execution from the UI: npm run compile, npm test, npm run build, git status, and git diff.
